# v0.1.1 reference score 評価

## 結論

v0.1.1 は、公式レンダリングをなるべく忠実に踏襲する版としてリリース可能。

ただし、公式レンダリングを完全踏襲した版とは表現しない。
完全踏襲を名乗るには、Draw.io と Mermaid の full compare 対象を 99 点以上へそろえる必要がある。

## 判定基準

- 95 点以上: 主要な図形、文字、接続線、全体構図が機能する状態。
- 99 点以上: 公式レンダリングを完全踏襲したと言える状態。
- 95 点未満: リリース前に修正する状態。

v0.1.1 では 95 点以上をリリース下限とし、99 点化は後続作業に分ける。

## 再評価結果

| 対象 | 全件 | 99 点未満 | 95〜99 点未満 | 95 点以下 | 最低点 |
| --- | ---: | ---: | ---: | ---: | ---: |
| Draw.io | 598 | 93 | 93 | 0 | 95.02247 |
| Mermaid | 117 | 19 | 19 | 0 | 95.58481 |
| 合計 | 715 | 112 | 112 | 0 | 95.02247 |

`wireframe_1` は切り抜き範囲の過大評価を修正し、94.33795 から 99.891848 へ改善した。
これにより、Draw.io / Mermaid の全量再評価で 95 点以下は 0 件になった。

## 視認評価

112 件の 95〜99 点未満を左右比較画像で確認した。
各比較画像は、左が公式、右が KDR。

確認した範囲では、次のリリース停止級の崩れは見当たらない。

- 図全体の空白化
- 主要ノードの欠落
- 主要エッジの欠落
- 大きな切り抜き崩壊
- 読めないほどの文字崩れ

残差の中心は次の種類。

- 文字位置、折り返し、余白の差
- 線の太さ、線の端、影の出方の差
- Draw.io のクラウド系テンプレートにおける小さいアイコンやラベルの差
- Mermaid の C4、ブロック図、ツリー、ベン図における配置差

## 残件の優先度

| スコア帯 | 件数 | 判断 |
| --- | ---: | --- |
| 95〜96 | 12 | 後続の 99 点化で最優先に扱う。 |
| 96〜97 | 14 | 視認差はあるが、リリース停止ではない。 |
| 97〜98 | 20 | 並べると差は分かるが、通常利用では許容範囲。 |
| 98〜99 | 66 | 主に微調整領域。 |

まず 95〜97 の 26 件から 99 点化するのが効率的。
